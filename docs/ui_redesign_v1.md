# Flashcutter UI 重构设计文档 v0.1

## 1. 背景与目标

Flashcutter 是一个面向设计、投放、视频运营人员的短视频广告生产工作台。它不是消费者剪辑器，也不是数据库后台。

本次 UI 重构的核心目标：

```text
让用户围绕“批量套模板生产广告变体”完成工作，而不是围绕素材表、模板表、任务表来回找功能。
```

第一主流程：

```text
一批已授权视频
→ 选择可复用的视频修改模板
→ 填写本次批次参数
→ 生成前预检
→ 加入渲染队列
→ 审核成片
→ 形成平台投放包
```

产品优先级：

```text
1. 可理解性
2. 易用性
3. 可回溯、可重试
4. 操作安全
5. 批量效率
```

第一版不追求极限效率，不做复杂快捷键和高密度批处理专家界面。

---

## 2. 已确认产品决策

### 2.1 主流程

第一主流程是：

```text
批量套模板生产广告变体
```

`仿制`是视频片段生产能力之一，但不是首页第一主流程。

### 2.2 模板权限

设计 / 运营人员可以自己创建模板。

模板 UI 必须把模板解释为：

```text
可复用的视频修改方法
```

而不是 JSON 配置或工程参数集合。

### 2.3 素材类型

系统需要统一管理：

```text
1. 种子视频
2. 视频片段
3. 图片素材：图片框、Logo、海报图、参考图
4. 配乐
5. 组件参考
```

AI 生成 clip 和用户上传的视频片段，在素材库中一视同仁。

### 2.4 队列与失败

失败任务只显示在：

```text
队列 / 失败处理
```

不进入审核页。

审核页只处理可审核成片：

```text
pending_review
approved
needs_changes
rejected
discarded
```

### 2.5 多用户隔离

队列数据必须严格按用户隔离。

```text
用户 A 只能看到自己的渲染任务、仿制任务、失败任务和队列位置。
用户 B 的任务不能影响用户 A 的队列数字、列表和提醒。
```

这需要补测试。

### 2.6 批量导入

广告样例文件和生产素材需要 UI 入口。

脚本导入继续保留，但试用用户应能在界面里完成批量导入。

### 2.7 一次性内容

严格执行：

```text
模板 = 可复用规则
生产批次参数 = 本次活动的一次性内容
```

一次性活动文案、价格、CTA、前贴片文件、后贴片文件、图片框文件、Logo 文件，不写入模板。

### 2.8 桌面优先

本轮设计桌面优先。

移动端只保证：

```text
能查看
不溢出
不遮挡
不崩坏
```

### 2.9 交付结果

试用客户最需要拿到的是：

```text
平台投放包
```

投放包包含已审核通过的视频、平台预检结果、素材元信息和后续推送状态。

## 4. 第一主流程：创建生产批次

生产工作台首页只突出一个核心动作：

```text
创建生产批次
```

点击后打开对话框，用户在一个连续流程中完成所有步骤，不需要凭记忆跳菜单。

### 4.1 对话框步骤

```text
1. 选择生产方式
2. 选择视频
3. 选择模板方法
4. 填写本次参数
5. 生成前预检
6. 加入队列
```

### 4.2 步骤 1：选择生产方式

以卡片展示，不用工程枚举名。

```text
一个视频生成多版
适合：同一个素材测试不同包装方式。

一批视频套同一个模板
适合：一批达人素材统一加前后贴片、图片框、Logo。

多个视频 x 多个模板
高级模式。适合一次生成平台规格矩阵。
```

默认推荐：

```text
一批视频套同一个模板
```

因为这是用户当前确认的关键新增流程。

### 4.3 步骤 2：选择视频

功能：

```text
1. 从素材库选择多个视频
2. 搜索文件名、标签、状态
3. 按比例筛选：9:16 / 1:1 / 16:9 / 其他
4. 显示时长、尺寸、是否可用
5. 支持批量导入入口
```

每个视频卡片显示：

```text
缩略预览
文件名
时长
比例
上传时间
状态
```

不可用视频不应只禁用按钮，必须说明原因：

```text
这个视频不能用于生产
原因：文件探测失败，无法读取时长和尺寸。
可操作：重新上传 / 查看失败详情
```

### 4.4 步骤 3：选择模板方法

模板按用途分组：

```text
前后贴片
图片框
品牌包装
尺寸适配
综合包装
```

模板卡片必须显示运营能理解的信息：

```text
模板名称
它会做什么
适合什么场景
本次需要补哪些素材
输出规格
审核重点
```

不要默认展示：

```text
schema_version
workflow json
width / height
内部 recipe_id
```

### 4.5 步骤 4：填写本次参数

根据模板 `runtime_fields` 动态生成表单。

字段类型映射：

| field_type | UI 控件 |
|---|---|
| asset | 素材选择器 |
| text | 文案输入框 |
| boolean | 开关 |
| choice | 单选或下拉 |
| number | 数字输入 |
| music | 配乐选择器 |

一次性内容只在这里填写：

```text
本次活动文案
本次 CTA
本次价格
本次前贴片
本次后贴片
本次图片框
本次 Logo
本次配乐
```

表单需要解释清楚字段归属：

```text
这些内容只保存到本次生产批次，不会写入模板。
```

### 4.6 步骤 5：生成前预检

预检不是“技术校验结果”，而是生产计划确认页。

预检顶部摘要：

```text
将创建 24 个任务
22 个可入队
2 个需要处理
预计渲染时间约 18 分钟
输出规格：竖版 9:16
```

预检项状态：

```text
可入队
可入队，有提醒
需处理
```

错误示例：

```text
这批任务还不能入队
原因：模板“竖版前后贴片 + 图片框”缺少必填图片框。
你可以：
1. 返回上一步选择图片框
2. 改用不需要图片框的模板
3. 暂存这批配置，稍后继续
```

预检必须支持：

```text
返回修改
重新预检
暂存草稿
确认入队
```

### 4.7 步骤 6：加入队列

成功后显示：

```text
已加入 24 个渲染任务
队列只显示你的任务
任务完成后会提醒你进入审核
```

按钮：

```text
查看队列
继续创建下一批
稍后处理
```

## 5. 素材库设计

### 5.1 素材库标签页

```text
种子视频
视频片段
图片素材
配乐
组件参考
```

### 5.2 种子视频

职责：

```text
管理可进入批量生产的原始视频。
```

核心动作：

```text
上传视频
批量导入
从 URL 导入
检测文字区域
查看分段
用于生产
```

### 5.3 视频片段

职责：

```text
管理 Hook / CTA / B-roll / Reaction / 其他可复用短片段。
```

来源：

```text
用户上传
仿制生成
系统导入
```

这三类在素材库中一视同仁，只在来源标签中区分。

### 5.4 图片素材

新增必要入口。

素材类型：

```text
图片框
Logo
贴片封面
海报图
参考图
其他
```

模板 runtime field 需要图片素材时，从这里选择。

### 5.5 组件参考

组件参考默认不进入生产。

默认权利状态：

```text
reference_only
```

UI 文案：

```text
这些素材只用于结构参考。确认授权前，不能作为生产素材使用。
```

## 6. 模板方法页设计

### 6.1 模板列表

默认展示可理解卡片：

```text
名称
类型
一句话用途
会执行的步骤
本次生产需要填写什么
输出规格
审核清单
最近使用
```

示例：

```text
竖版前后贴片 + 图片框
类型：前后贴片 / 图片框
会执行：前贴片 → 竖版适配 → 套图片框 → 后贴片
本次需要：图片框；可选前贴片、后贴片
输出：竖版 9:16
```

### 6.2 创建模板向导

创建模板不从 JSON 开始。

步骤：

```text
1. 选择模板类型
2. 添加修改步骤
3. 定义本次生产要填写的参数
4. 选择输出规格
5. 设置审核清单
6. 预览与保存
```

模板类型：

```text
前后贴片
图片框
品牌包装
尺寸适配
综合包装
```

修改步骤使用自然语言：

```text
在视频前添加前贴片
在视频后添加后贴片
给视频套图片框
添加 Logo 水印
遮盖底部字幕区
添加可替换文案
替换配乐
调整为平台规格
```

### 6.3 输出规格

用户不能手填宽高。

只显示：

```text
竖版 9:16，填满画面
竖版 9:16，保留完整画面
方版 1:1
横版 16:9
保持原尺寸
```

工程映射隐藏在系统内部。

### 6.4 高级 JSON

高级 JSON 只放在折叠区。

文案：

```text
高级配置。通常不需要编辑。
```

## 7. 队列与失败处理

### 7.1 页面职责

队列页统一展示：

```text
渲染任务
仿制任务
导入任务
失败任务
```

但所有数据必须按当前用户隔离。

### 7.2 队列分组

```text
运行中
排队中
已完成，待审核
失败，可处理
已取消
```

### 7.3 任务卡片字段

```text
任务名
所属批次
任务类型：渲染 / 仿制 / 导入
当前阶段
已等待 / 已运行
进度
失败步骤
下一步动作
```

### 7.4 失败状态文案规范

不能显示：

```text
Failed to fetch
Render failed
Invalid params
```

必须显示：

```text
这一步没有完成
发生在：生成前预检 / 入队 / 渲染 / 后处理 / 入库
原因：可读原因
你可以：返回修改 / 重试 / 复制为新批次 / 查看技术详情
```

示例：

```text
这个任务没有生成成片
发生在：渲染输出校验
原因：输出 fps 为 29，但模板要求 30。
你可以：
1. 使用“保持原帧率”模板重新生成
2. 复制任务并降低输出规格
3. 查看技术详情
```

### 7.5 队列完成提醒

当用户不在审核页，并且自己的队列从运行中变为完成后，弹出提醒：

```text
任务已完成，请审核
```

按钮：

```text
去审核
稍后处理
```

提醒只能基于当前用户的任务。

## 8. 多用户队列隔离测试

需要新增后端测试，覆盖渲染队列和仿制队列。

### 8.1 测试目标

```text
用户只能看到自己的队列、失败、任务事件、队列位置和完成提醒基础数据。
```

### 8.2 建议测试用例

```text
test_user_cannot_list_other_users_generation_tasks
test_user_cannot_list_other_users_task_events
test_user_cannot_list_other_users_review_outputs
test_ai_clone_jobs_are_user_scoped
test_ai_clone_queue_positions_are_user_scoped
test_queue_completion_counts_are_user_scoped
test_failed_tasks_are_not_visible_in_review_outputs
```

### 8.3 队列位置规则

用户 A 有 3 个任务，用户 B 有 2 个任务。

用户 A 看到：

```text
第 1 位
第 2 位
第 3 位
```

用户 B 看到：

```text
第 1 位
第 2 位
```

不能显示全局队列位置。

### 8.4 UI 规则

队列页文案：

```text
这里只显示你的任务。
```

不要展示全局排队人数，除非后续明确设计成共享 GPU 资源池并做权限说明。

## 9. 审核页设计

### 9.1 页面职责

审核页只处理已产出的成片。

默认按生产批次分组：

```text
批次
  ├── 变体 1
  ├── 变体 2
  └── 变体 3
```

### 9.2 审核动作

```text
通过
要求修改
丢弃
拒绝
```

要求修改必须结构化：

```text
修改类型
修改对象
具体要求
优先级
备注
```

### 9.3 审核到投放包

审核通过后，批次顶部显示：

```text
已通过 N 条
创建平台投放包
```

按钮：

```text
创建投放包
打包下载
```

## 12. 视觉方向

### 12.1 整体风格

```text
桌面生产工具
克制、清晰、可靠
信息密度中等偏高
不做营销页
不做大 hero
不做装饰性渐变背景
```

### 12.2 推荐布局

```text
左侧导航：220-240px
顶部状态栏：当前页面标题 + 全局刷新 + 用户信息
主工作区：流程卡片 / 表格 / 列表
右侧抽屉：详情、预检、失败原因、技术详情
```

### 12.3 控件规则

```text
模板类型：分段控件或标签筛选
生产方式：大卡片单选
输出规格：预设卡片或下拉
素材选择：可搜索选择器 + 缩略预览
批量操作：明确确认弹窗
失败详情：功能化提示框
```

## 13. 实施分期

### Phase 1：信息架构和主流程

```text
1. 重命名导航
2. 新建生产工作台
3. 重构创建生产批次对话框
4. 预检结果改成可操作状态
5. 队列完成提醒改为用户级
```

### Phase 2：素材库统一

```text
1. 合并种子视频、视频片段、图片素材、配乐、组件参考的入口
2. 增加图片素材上传与选择
3. 增加批量导入 UI
4. 仿制入口降级为视频片段的创建方式之一
```

### Phase 3：模板方法重构

```text
1. 模板列表改为用途卡片
2. 创建模板向导
3. 输出规格预设化
4. 高级 JSON 折叠
5. 严格拆分模板规则与批次参数
```

### Phase 4：队列、失败和多用户隔离

```text
1. 队列页按用户过滤
2. 失败任务功能化处理
3. 补多用户队列隔离测试
4. 失败任务不进入审核页
```

### Phase 5：审核到投放包

```text
1. 审核页按批次决策
2. 通过后创建投放包
3. 平台规格预检
4. 投放包下载 / 模拟推送
```
