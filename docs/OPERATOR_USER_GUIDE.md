# Flashcutter 试用用户使用说明

这份说明给试用用户使用，不解释系统架构和技术原理，只说明日常怎么操作。

## 1. Flashcutter 是做什么的

Flashcutter 用来把一条已经授权使用的原始广告视频，批量做成多个短视频广告变体。

你可以把它理解成一个广告视频生产工作台：

```text
上传原始视频
选择扩量方案包
批量生成变体
人工审核
下载可用成片
```

Flashcutter 不是通用剪辑软件，也不是投放平台。现阶段它主要帮助你更快做出一批可检查、可复盘、可交付的广告素材。

## 2. 使用前准备

请先准备：

```text
1. 已确认可以使用的原始视频。
2. 需要测试的广告方向或素材需求。
3. 可选：自己的配乐文件。
4. 可选：可重复使用的视频片段，例如开场、结尾、产品特写。
```

重要：素材授权请在上传前确认。Flashcutter 不负责判断某条素材是否可商用。

## 3. 登录

打开试用地址后，用分配给你的手机号和密码登录。

如果需要修改密码，可以在左侧用户区域展开账号操作，输入当前密码和新密码后提交。

## 4. 上传原始视频

进入「素材」页面。

你可以：

```text
上传本地视频
或导入视频 URL
```

上传后的原始视频会作为后续生产广告变体的种子视频。建议文件命名能让人看懂，例如：

```text
奶茶新品-窗边实拍-竖版.mp4
```

这样审核和下载时更容易识别。

## 5. 准备配乐

进入「配乐库」页面。

配乐分两类：

```text
系统公用库：系统预置，所有试用用户可见。
我的私有库：你自己上传，只给当前账号使用。
```

你可以先试听系统配乐。如果需要使用自己的音乐，上传到私有库即可。

当前试用版本的规则很简单：方案包选择了配乐后，生成视频会用这首配乐替换原视频声音。

## 6. 准备视频片段

进入「视频片段」页面。

这里放可以重复使用的短视频片段，例如：

```text
开场 Hook
结尾 CTA
产品特写
氛围 B-roll
反应镜头
```

你可以上传自己的片段，并填写标题、类型、标签和说明。后续 AI 仿制生成的片段也会进入这里，和手动上传的片段一起管理。

## 7. 查看和调整扩量方案包

进入「模板方法」页面。

普通生产时，你面对的是“扩量方案包”，不是底层剪辑参数。方案包描述的是这批视频要达到什么扩量目标，例如：

```text
Hook 扩量
基础安全扩量
产品证据强化
CTA 强化
平台去重 / 疲劳缓解
```

方案包卡片会显示：

```text
适合什么素材
不适合什么素材
预计会执行什么
需要额外准备哪些素材或字段
输出规格
审核重点
当前账号历史表现和常见修改原因
```

内置方案包不能直接覆盖。你可以基于内置方案包保存一个自己的副本，再修改。

普通编辑默认只需要选择方案强度：

```text
保守：少改动，优先稳定过审
标准：适度节奏和画面增强
激进：更明显的节奏和开场冲击
```

高级参数仍然保留给内部运营，例如：

```text
画面比例和尺寸
剪辑节奏
画面叠字
配乐
画面风格
左右镜像
转场和质感
审核重点
```

如果不确定某个方案是否合适，建议先用一条短视频跑少量变体，再看审核结果。

## 8. 创建一批广告变体

进入「变体生成」或生产相关页面。

新手优先使用页面顶部的「强开场批量生成」：

```text
1. 选择一条已授权的种子视频。
2. 填写产品名、卖点、禁用词和生成数量。
3. 点击“生成 AI 建议版”，系统会给出多条前三秒开场文字。
4. 你可以直接编辑，也可以用自己的每行一条文案替换 AI 建议。
5. 运行预检，通过后点击“一键入队”。
```

这个入口会自动使用强开场方案包，不需要先理解模板参数。适合先把一条素材扩成几十条开场不同的可审核变体。

生产向导适合更复杂的批量组合：

```text
1. 选择一个原始视频。
2. 查看素材体检和推荐方案包。
3. 选择一个或多个方案包，或点击“选择推荐方案包”。
4. 检查本次生产设置。
5. 运行预检，通过后入队。
```

推荐只负责帮你减少选择成本，真正能不能批量生产仍以预检为准。推荐分为：

```text
推荐：适合当前素材先跑。
可尝试：可以小批量试验。
谨慎：需要补素材、字段，或有裁切/时长风险。
```

只创建任务：先把任务列出来，稍后再手动运行。

创建并加入渲染队列：立即排队开始生成视频。

建议试用时先选择 2-3 个方案包，确认效果和速度后再扩大批量。

## 9. 查看队列和失败任务

进入「队列」或「任务」页面。

这里可以看到：

```text
正在排队的任务
正在渲染的任务
已完成任务
失败任务和失败原因
```

失败任务不会进入审核页。需要先在队列里查看原因，再决定是否重试或调整方案包、本次参数。

## 10. 审核生成结果

进入「审核」页面。

审核页按生产批次组织结果。一个批次通常对应：

```text
一次原始视频
一组方案包
一批生成出来的广告变体
```

批次名称会尽量包含时间和原始素材名，方便你识别。

你可以逐条查看变体，并做决定：

```text
通过：这个视频可以交付或进入后续投放准备。
要求修改：这个方向可用，但还需要改。
拒绝：这个视频不适合继续使用。
丢弃：不进入后续处理。
```

如果选择「要求修改」，优先点击结构化修改原因，例如：

```text
开场弱
产品太晚
字幕难读
配乐不合适
节奏太慢
裁切挡主体
CTA 不明显
变化太小
```

你也可以继续编辑具体要求。系统会根据当前批次的结构化反馈给出下一轮建议，例如改用 Hook 扩量、产品证据强化、CTA 强化，或降低裁切风险。

## 11. 标记整个批次需要再生产

有时单条视频的问题不大，但整批都需要重新来一轮，例如：

```text
所有视频节奏都太慢。
配乐整体不适合。
叠字方向需要统一重写。
方案包选错了。
```

这时可以把整个批次标记为需要再生产。

系统会保留原批次和后续修改轮次的关系，方便之后追溯“为什么改、改了几轮”。

## 12. 打包下载过审视频

当一个批次里已经有通过审核的视频，可以在审核页使用「打包下载」。

打包内容包括：

```text
原始种子视频
这个批次里所有已通过的视频
```

下载前系统会提示预计文件大小。文件较大时请确认网络和磁盘空间足够。

这个 zip 包可以作为连接广告平台之前的过渡交付物，直接发给合作方或投放同事检查。

## 13. AI 仿制生成

AI 相关功能是独立实验能力，不是当前主生产流程的必经步骤。

它的目标是：用一张参考图或一段参考视频，生成一个相似风格的新短片段，然后进入「视频片段」库。

现阶段请把它看作补充素材的方法，而不是自动生成完整广告的方法。

## 14. 建议的试用顺序

第一次试用建议按这个顺序：

```text
1. 登录。
2. 上传 1 条已授权原始视频。
3. 试听系统配乐。
4. 选择 2-3 个推荐方案包生成变体。
5. 在队列页确认任务完成。
6. 在审核页逐条审核。
7. 通过 1-2 条视频。
8. 打包下载。
9. 记录需要修改的问题。
```

第二次试用再尝试：

```text
上传自己的配乐
上传自己的视频片段
修改方案包副本
标记批次需要再生产
```

## 15. 反馈时请尽量说明

如果遇到问题，请尽量记录：

```text
发生在哪个页面
原始视频名称
批次名称或任务编号
你点击了什么
你期待看到什么
实际发生了什么
是否有错误提示
```

越具体，越容易定位和修复。
