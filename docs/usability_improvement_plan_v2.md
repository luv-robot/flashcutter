# Flashcutter 易用性设计改进计划 v0.2

更新日期：2026-06-02

## 1. 参考来源


本版不替代 `ui_redesign_v1.md` 和 `template_system_v3.md`。它的用途是把设计文档和试用用户手册压缩成下一轮可执行的易用性落地计划。

## 2. 设计判断

Flashcutter 现在的主用户不是剪辑师个人，而是设计、投放、视频运营人员。他们的核心目标不是“编辑一个视频”，而是：

```text
把已授权素材快速变成一批可审核、可复盘、可下载或投放准备的短视频广告变体。
```

因此易用性优先级应保持：

```text
1. 新用户第一次试用能走通。
2. 批量生产不要求用户理解工程字段。
3. 失败和预检必须可操作。
4. 审核反馈能进入下一轮生产。
5. 权利、审核和投放准备语言不能被弱化。
```

当前前端已经具备新版信息架构雏形：

```text
生产工作台
素材库
模板方法
队列与失败
审核
投放包
```

下一步重点不是继续扩导航，而是让每个页面都回答一个问题：

```text
我现在该做什么，为什么不能继续，下一步去哪？
```

## 3. 试用用户路径

使用手册中的第一次试用路径应成为产品内默认引导：

```text
登录
上传 1 条已授权原始视频
试听系统配乐
选择 2-3 个模板生成变体
在队列页确认完成
在审核页逐条审核
通过 1-2 条视频
打包下载
记录修改问题
```

现有界面不应假设用户已经读完手册。生产工作台需要把这条路径变成“首次试用清单”和空状态 CTA。

## 4. 主要易用性问题

| 区域 | 现状 | 易用性风险 | 改进方向 |
|---|---|---|---|
| 登录 | 前端仍提供创建账号入口，但线上注册关闭且未接短信 | 试用用户会误以为能自助注册 | 暂时隐藏注册入口，改为“使用分配账号登录”；后续接短信或邀请码再开放 |
| 生产工作台 | 有概览和创建入口 | 首次用户不知道先准备哪些素材 | 加首次试用清单、缺失项提示和一键跳转 |
| 素材库 | 已合并入口，图片素材仍未接入 | 模板需要图片框/Logo 时会卡住 | 先接图片素材上传和选择，明确权利状态 |
| 配乐库 | 系统曲库和私有曲库已分 | 用户不知道配乐会替换原声 | 在模板和配乐处同时提示“替换原视频声音” |
| 模板方法 | 已有可视表单，但仍展示 JSON | 运营误以为必须懂 JSON | 默认展示用途卡片和自然语言步骤，高级 JSON 折叠到最后 |
| 创建生产批次 | 已有 6 步向导和多种模式 | 缺少更强的“为什么不能入队”解释 | 预检结果按 ready/warning/blocked 分组，并给每个 blocked 项提供返回动作 |
| 队列与失败 | 已按状态分组 | 失败原因若来自技术错误仍难懂 | 建立错误文案映射：发生在哪一步、原因、可做什么 |
| 审核 | 已按批次/素材组织 | 批次再生产和结构化修改还需要更清晰 | 把“要求修改”做成结构化表单，批次级再生产显示版本关系 |
| 投放包 | 有页面雏形 | 用户最想拿到交付物，但路径仍弱 | 审核通过后直接显示打包下载和创建投放包 |
| AI 仿制 | 已在视频片段页 | 容易被误解为主生产流程 | 保持为“创建视频片段”的入口，不放在主流程第一位 |

## 5. 页面级改进方案

### 5.1 登录和账号

短期：

```text
1. 隐藏“创建账号”切换按钮。
2. 登录页文案改为：请使用分配的手机号和密码登录。
3. 登录错误分三类显示：账号不存在、密码错误、服务不可用。
4. 保留登录后修改密码。
```

中期：

```text
1. 增加邀请码注册。
2. 或接短信网关后增加验证码注册。
3. 注册开放前，后端继续保持 FLASHCUTTER_ALLOW_REGISTRATION=false。
```

验收：

```text
试用用户不会在无短信网关状态下尝试自助注册。
```

### 5.2 生产工作台

新增首次试用清单：

```text
1. 已登录
2. 已有可用种子视频
3. 系统配乐可试听
4. 至少有 1 个可用模板方法
5. 当前没有阻塞失败任务
6. 有待审核成片时提示去审核
```

每个缺失项给明确 CTA：

```text
上传视频
打开配乐库
选择模板方法
查看失败任务
去审核
```

工作台指标保留，但必须和动作绑定：

```text
可用素材 -> 去素材库
模板方法 -> 去模板方法
队列中 -> 去队列与失败
待审核 -> 去审核
已通过 -> 去投放包
```

### 5.3 素材库

素材库继续使用现有标签：

```text
种子视频
视频片段
图片素材
配乐
组件参考
```

优先补齐图片素材：

```text
上传图片
填写类型：图片框 / Logo / 贴片封面 / 海报图 / 参考图 / 其他
填写权利状态：已授权 / 待确认 / 仅参考
可被模板 runtime field 选择
```

权利文案必须固定显示：

```text
素材授权请在上传前确认。Flashcutter 不负责判断某条素材是否可商用。
```

组件参考默认不可进入生产：

```text
这些素材只用于结构参考。确认授权前，不能作为生产素材使用。
```

### 5.4 创建生产批次

保留现有 6 步向导：

```text
选择生产方式
选择视频
选择模板
填写本次参数
预检
入队
```

把三种生产方式固定为业务语言：

```text
一个视频生成多版
一批视频套同一个模板
多个视频 x 多个模板
```

预检结果必须拆成：

```text
ready：可以入队
warning：可以入队，但需要用户确认风险
blocked：不能入队，必须补充信息或改模板
```

每个 blocked 项显示：

```text
缺什么
发生在哪个视频和模板
为什么会阻塞
可以点击去哪一步修
```

示例：

```text
不能入队
视频：奶茶新品-窗边实拍-竖版.mp4
模板：竖版前后贴片 + 图片框
原因：这个模板需要图片框，但本次参数未选择图片框。
你可以：返回填写本次参数 / 改用其他模板 / 暂存草稿
```

### 5.5 模板方法

模板列表默认不显示 JSON。

模板卡片显示：

```text
模板名称
类型
一句话用途
会执行的步骤
本次生产需要填写什么
输出规格
审核清单
最近使用
```

模板编辑分层：

```text
基础信息
修改步骤
本次参数
输出规格
审核清单
高级 JSON
```

输出规格只允许选择业务预设：

```text
竖版 9:16，填满画面
竖版 9:16，保留完整画面
方版 1:1
横版 16:9
保持原尺寸
```

不要让普通运营手填：

```text
width
height
fit
fps
codec
```

### 5.6 队列与失败

队列页只显示当前用户任务，并明确文案：

```text
这里只显示你的任务。
```

任务分组保留：

```text
运行中
排队中
已完成，待审核
失败，可处理
已取消
```

失败任务必须显示：

```text
当前处境
失败步骤
可读原因
下一步动作
技术详情折叠
```

禁止只显示：

```text
Failed to fetch
Render failed
Invalid params
```

队列完成提醒保留，但要确保只基于当前用户。

### 5.7 审核

审核页继续只处理已产出的成片，不混入失败任务。

审核动作：

```text
通过
要求修改
丢弃
拒绝
```

要求修改改成结构化：

```text
修改类型：画面 / 文案 / 配乐 / 模板 / 素材选择 / 其他
修改对象：开场文字 / 产品镜头 / CTA / 背景音乐等
具体要求
优先级
备注
```

批次级再生产需要显示版本关系：

```text
原批次 R1
本次修改 R2
修改原因
继承了哪些视频、模板和本次参数
```

审核通过后，批次顶部直接显示：

```text
已通过 N 条
打包下载
创建投放包
```

### 5.8 投放包

短期先把投放包作为交付和预检工具，不急于真实平台 API。

流程：

```text
审核页选择 approved 视频
创建投放包
填写平台、商品名、落地页、行业、素材标题
预检
下载 zip 或 mock 推送
```

预检必须拦截：

```text
未审核视频
缺少落地页
尺寸不符合目标平台
缺少素材标题
缺少权利确认记录
```

安全边界：

```text
不提供绕过审核、规避检测、伪装素材的功能。
```

### 5.9 AI 仿制和视频片段

AI 仿制继续作为视频片段库的创建方式之一：

```text
视频片段库
  上传片段
  上传参考图
  仿制生成
```

不在首页主流程中突出 AI。

仿制对话框必须自动选择 workflow：

```text
图片参考 -> image_clone_video_v1
视频参考 -> video_clone_clip_v1
```

错误提示使用功能化文案：

```text
参考图太大：请压缩到 10MB 以内或改用 JPG/PNG。
参考视频太长：请截取 3-8 秒片段后再仿制。
生成服务暂时离线：任务未扣费，请稍后重试。
```

## 6. 优先级路线图

### P0：试用可用性修补，1-2 天

目标：新用户不会在第一屏卡住。

```text
1. 隐藏注册入口，改成分配账号登录文案。
2. 生产工作台增加首次试用清单。
3. 所有空状态增加下一步 CTA。
4. 配乐库显示系统曲目数量和试听入口。
5. 首页、素材库、生产向导补充“已授权素材”提示。
```

验收：

```text
用户不读手册，也能知道先上传视频、再创建生产批次。
```

### P1：生产批次向导打磨，3-5 天

目标：从“一个视频 x 2-3 模板”到入队不需要解释。

```text
1. 预检结果按 ready / warning / blocked 分组。
2. blocked 项提供返回动作。
3. 批次命名自动带素材名、模板组和时间。
4. 加入“暂存草稿”。
5. 入队完成后明确提示去队列或稍后审核。
```

验收：

```text
试用用户能用 1 条视频和 2-3 个模板完成一次生产，不接触 JSON。
```

### P2：素材库和模板方法补齐，1 周

目标：把“模板需要什么素材”变成可选择对象。

```text
1. 接入图片素材上传和选择。
2. 模板卡片显示 required_runtime_fields。
3. 模板编辑默认隐藏 JSON。
4. 输出规格改成预设选择。
5. 配乐选择处显示系统曲库和私有曲库来源。
```

验收：

```text
用户能创建或编辑一个“前后贴片 + 图片框”模板，并在生产批次里填写图片框、前贴片、配乐等本次参数。
```

### P3：队列、失败和审核闭环，1 周

目标：失败可以处理，审核反馈能进入下一轮。

```text
1. 队列接口和列表严格按用户隔离。
2. 失败任务建立错误码到运营文案的映射。
3. 失败任务支持复制为新批次或返回原批次参数。
4. 审核“要求修改”改成结构化表单。
5. 批次再生产显示 R1 / R2 关系。
```

验收：

```text
失败任务不会进入审核页；审核页能记录结构化修改意见；批次能基于修改意见再生产。
```

### P4：审核到投放包，1 周

目标：让试用客户拿到明确交付物。

```text
1. 审核通过后显示打包下载和创建投放包。
2. 投放包预检平台字段。
3. 支持 zip 下载：原始 seed、approved 视频、元信息、审核记录。
4. 支持 mock 平台推送状态。
5. 支持导出 CSV。
```

验收：

```text
运营能从一个批次选择 approved 视频，生成可交付的投放包，并看到平台预检结果。
```

### P5：AI 仿制降噪和稳定化，主流程稳定后

目标：让 AI 仿制成为补充素材能力，而不是打断主流程。

```text
1. 仿制入口收进视频片段库。
2. 自动 workflow 选择。
3. 生成结果进入统一视频片段库。
4. 失败退款或释放冻结 credits。
5. 任务状态和队列只显示当前用户。
```

验收：

```text
用户能从参考图或参考视频生成一个视频片段，并在生产模板中把它当作普通片段使用。
```

## 7. 度量指标

试用可用性指标：

```text
1. 第一次登录后 15 分钟内完成一次 1 视频 x 2 模板生产。
2. 第一次试用无需打开 README 或让开发解释页面含义。
3. 预检 blocked 项中，用户能明确知道下一步去哪。
4. 审核页至少通过 1 条视频并完成打包下载。
5. 注册入口不再造成误解。
```

运营效率指标：

```text
1. 创建生产批次平均点击不超过 12 次。
2. 模板选择不需要查看 JSON。
3. 失败任务 80% 能通过 UI 推荐动作处理。
4. 待审核成片能按批次、素材、模板快速定位。
```

安全和合规指标：

```text
1. 所有生产素材上传入口都有授权提示。
2. 组件参考默认不可生产使用。
3. 只有 approved 成片可以进入投放包。
4. 不显示或暗示绕过平台审核的功能。
```

## 8. 下一轮建议开发任务

建议从 P0 开始拆 4 个小 PR：

```text
PR A：登录与首次试用清单
PR B：生产向导预检可操作化
PR C：模板卡片和高级 JSON 折叠
PR D：审核到打包下载入口强化
```

每个 PR 都应包含：

```text
1. UI 改动
2. 空状态和错误状态
3. 至少一条 smoke 或后端测试
4. 用户手册同步更新
```

## 9. 非目标

本轮不做：

```text
1. 复杂快捷键专家界面。
2. 移动端完整生产流程。
3. 真实广告平台 API 全量接入。
4. 开放短信注册，除非短信网关和风控已经准备好。
5. 任何绕过平台检测或审核的功能。
```

## 10. 结论

这版易用性改进的核心不是新增更多页面，而是把当前已经具备的生产、素材、模板、队列、审核、投放包页面串成一条用户不用记忆的路径：

```text
准备素材
创建生产批次
预检并入队
处理失败
审核成片
打包交付
```

只要 P0-P2 完成，试用客户就能比较自然地完成第一轮真实素材试生产。P3-P4 完成后，Flashcutter 才会从“能生成视频的工具”变成“能支撑广告生产复盘和交付的工作台”。
